MediConnect - Advanced Viva Questions

These are more specific, deeper technical questions that examiners might ask to test your mastery of the project.

1. Data & State Management

Q: How do you handle "Race Conditions" when two people try to book the same slot simultaneously?
I use Prisma Transactions (or database-level constraints). The Availability model has a unique constraint on [doctorId, dayOfWeek, startTime]. This ensures that the database itself prevents duplicate bookings for the exact same moment if another operation is already pending.
Q: Why use Zod for validation instead of simple if/else checks?
Zod provides a Single Source of Truth for data shapes. It automatically throws errors if a user tries to send a string where a number is expected (like for 'fee' or 'experience'), preventing corrupted data from ever reaching the PostgreSQL database.

2. Authentication & Middleware

Q: What is the difference between a Session and a JWT in your project?
My project uses JWT (JSON Web Tokens) via NextAuth. This is 'stateless', meaning the server doesn't have to keep a list of logged-in users. The token contains the User's ID and Role; whenever they visit a page, the middleware simply decrypts the token to verify who they are.
Q: How do you protect the system from Brute Force attacks on the login page?
NextAuth has built-in protection, but I also use hashed passwords with Bcrypt. Even if an attacker saw the database, they couldn't read the passwords. For further protection, I could implement a 'blocked' status in the User model after X failed attempts (feature already present in the schema).

3. Performance & Scaling

Q: Explain "Next.js Server Components" (RSC) and why they are used here.
RSCs allow the dashboard data (like doctor lists) to be fetched directly on the server. This means no "Loading..." spinners for the user, better security (API keys aren't exposed to the browser), and a much faster "First Contentful Paint."
Q: How would your app handle 1 million users?
Technically, I would:
  • Add Redis for caching frequently searched doctors.
  • Use Database Indexing on the email and specialization columns to speed up searches.
  • Deploy on a platform like Vercel or AWS that scales automatically.

4. The "Social" Questions

Q: What was the hardest technical decision you had to make?
Choosing whether to make the Symptom Checker purely AI-driven or Rule-based. I decided on a hybrid approach (mapping keywords to specializations) because in medical apps, Accuracy is more important than creative AI responses.
Q: How do you ensure the privacy of Patient data?
Data is stored in a secured PostgreSQL instance. Only the specific Doctor booked and the System Admin can see appointment details. I used role-based filtering in Prisma queries (e.g., where: { patientId: session.user.id }) to ensure users can't see each other's history.
